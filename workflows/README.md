# ワークフロー (Workflows)

このディレクトリは、DeepCodeの自律エージェントの思考プロセスとタスク実行フローを定義するコアロジックを格納しています。

## ディレクトリ構造

| ファイル名 | 説明 |
|:---|:---|
| `agent_orchestration_engine.py` | **オーケストレーター**。タスクの全体的な制御、エージェントの調整、ワークフローの分岐判断を行うメインエンジンです。 |
| `code_implementation_workflow.py` | コード生成・実装に特化したワークフロー定義。要件定義、コーディング、テストのサイクルを管理します。 |
| `code_implementation_workflow_index.py` | コードインデックス作成と検索を含む実装ワークフローのバリエーション。 |
| `codebase_index_workflow.py` | プロジェクト全体のコードベースを解析し、検索用インデックスを構築するためのワークフロー。 |
| `agents/` | 個別のエージェント（プランナー、コーダー、テスターなど）の具体的な実装を含むサブディレクトリ。 |

## 役割

DeepCodeの「頭脳」にあたる部分です。単なるツール呼び出しではなく、複雑なタスクをどのように分解し、どの順序でエージェントを動かし、結果を検証するかという「自律的な振る舞い」がここで定義されています。

## アーキテクチャ

ワークフローは通常、以下のパターンで構成されます：
1.  **入力解析**: ユーザーの要求や論文を理解。
2.  **プランニング**: タスクをステップに分解。
3.  **実行**: 各ステップに対応する専門エージェントまたはツールを呼び出し。
4.  **検証・修正**: 結果を評価し、必要に応じて再実行や修正を行うループ。
